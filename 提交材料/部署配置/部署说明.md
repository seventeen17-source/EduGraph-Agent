# EduGraph-Agent 部署说明

两种部署方式任选其一。推荐评审/演示环境使用 **方式 A（本地直跑）**，更直观且与开发环境一致。

---

## 方式 A：本地直跑（推荐）

### 1. 前置软件

| 软件 | 版本 | 说明 |
|------|------|------|
| Python | 3.11+ | 后端运行时 |
| Node.js | 18+ | 前端运行时 |
| Neo4j | 5.x Community | 本地安装（Neo4j Desktop 亦可） |

### 2. 启动 Neo4j 并导入数据

1. 启动 Neo4j，确认 `bolt://localhost:7687` 可连，账号 `neo4j`、密码 `edugraph123`（首次启动请把默认密码改为此值，或在 `.env` 里改成你的密码）；
2. 在项目根目录执行导入：

```bash
# 先做本地校验（不写库）
python Scripts/import_to_neo4j.py --dry-run

# 正式导入（清空重建）
python Scripts/import_to_neo4j.py --uri bolt://localhost:7687 --user neo4j --password edugraph123 --database neo4j --drop-existing
```

导入内容：Course / Chapter / KnowledgePoint / Source / Exercise / DocumentChunk / CodeCase / FAQ / Misconception 及全部关系边（共 370 节点、1067 边）。

### 3. 配置环境变量

```bash
# 复制模板并填入你的 LLM / Embedding API Key
cp 提交材料/部署配置/.env.example backend/.env
```

必填：`LLM_API_KEY`（及配套 `LLM_BASE_URL`/`LLM_MODEL`）、`EMBEDDING_API_KEY`。讯飞图片生成为可选项，不配置仅影响"教学图片"这一类资源。

### 4. 启动后端

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --port 8000
```

验证：`http://127.0.0.1:8000/health` → `{"status":"ok","neo4j":"ok"}`；Swagger UI：`http://127.0.0.1:8000/docs`。

### 5. 启动前端

```bash
cd frontend
npm install
npm run dev
```

打开 `http://127.0.0.1:5173`，用演示账号登录（`demo@edugraph.local` / `demo123`）。

### 6. 冒烟验证（可选）

```bash
cd backend
python tests/test_e2e_smoke.py
```

## 方式 B：Docker Compose

> 适合快速在干净机器上拉起全套服务（Neo4j + 后端 + 前端）。

### 1. 准备

```bash
cd 提交材料/部署配置
cp .env.example .env      # 填入 LLM / Embedding API Key
```

### 2. 启动

```bash
docker compose up -d --build
```

服务：

| 服务 | 地址 |
|------|------|
| Neo4j Browser | http://localhost:7474（neo4j / edugraph123） |
| 后端 API | http://localhost:8000（/docs 为 Swagger） |
| 前端 | http://localhost:5173 |

### 3. 首次导入图谱数据

容器全部就绪后执行一次：

```bash
docker exec edugraph-backend python /app/Scripts/import_to_neo4j.py \
  --uri bolt://neo4j:7687 --user neo4j --password edugraph123 --drop-existing
```

### 4. 停止 / 清理

```bash
docker compose down          # 停止（保留 Neo4j 数据卷）
docker compose down -v       # 停止并删除数据卷
```

## 常见部署问题

| 问题 | 处理 |
|------|------|
| `/health` 显示 neo4j 异常 | 确认 Neo4j 已启动、账号密码与 `.env` 一致；Docker 方式下 URI 应为 `bolt://neo4j:7687` |
| 搜索知识点无结果 | 未导入数据，执行导入脚本 |
| LLM 相关接口返回 unavailable | `.env` 的 `LLM_API_KEY` 缺失/无效/欠费 |
| 前端 Network Error | 后端未启动或端口被占；前端默认请求 `http://<同主机>:8000`，可用 `VITE_API_BASE_URL` 覆盖 |
| Windows PowerShell npm 报执行策略错误 | 用 `npm.cmd` 代替 `npm` |
