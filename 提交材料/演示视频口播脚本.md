## 二、完整口播脚本

---

### ⏱ 段落①：开场 + 登录（0:00 - 0:30 / 30秒 / 占比7%）

**画面：** 打开浏览器 → 地址栏输入 `http://127.0.0.1:5174` → 看到登录页 → 点击「以演示账号体验」→ 自动登录成功 → 跳转到学习助手页

**口播：**

> "各位好，今天为大家演示 **EduGraph-Agent**——一个基于知识图谱与多智能体协同的个性化学习平台。我们以《机器学习》课程为载体，构建了诊断、讲解、练习、反馈、规划的完整学习闭环。
>
> 点击「以演示账号体验」，一键登录，系统已预置了完整的学习画像数据，接下来我们直接进入核心功能。"

**操作要点：** 鼠标点击干净利落，登录后等待页面完全加载再开始下一段。

---

### ⏱ 段落②：画像对话（0:30 - 1:10 / 40秒 / 占比10%）

**画面：** 左侧导航点击「画像对话」→ 展示已有画像对话记录 → 顶部显示画像完整度 → 切换到「画像面板」→ 展示 8 维画像内容（学习目标、基础水平、认知风格、资源偏好等）→ 页面下滑展示"缺失维度"提示和薄弱知识点列表 → 切回学习助手页，看左侧学习概况卡片

**口播：**

> "首次使用系统时，会通过自然语言对话来构建你的学习画像。演示账号已经完成了这个流程——可以看到画像完整度达到 80% 以上，覆盖了 8 个维度：学习目标、知识基础、认知风格、时间投入、资源偏好等等。
>
> 进入画像面板，可以查看 8 个维度的详细信息。注意——画像数据通过对话自然提取，系统会从每一次对话、练习和反馈中持续挖掘学习特征，因此无需手动编辑表单。部分维度如『能力水平』在数据不足时会显示『暂无』，持续对话后会自动填充。画像越完整，后续的知识讲解、资源推荐和学习路径就越贴合你的个人情况。
>
> 回到学习助手页，左侧面板会实时显示你的学习概况——完整度、偏好资源类型、需要加强的知识点。系统所有的推荐决策都基于这些画像数据。"

**操作要点：** 在画像面板页面稍微停留，让鼠标在几个关键字段上划过，展示数据丰富度。可以指向显示"暂无"的字段并自然带过。

### ⏱ 段落③：⭐ 学习助手对话（1:10 - 3:30 / 2分20秒 / 占比33%）

**——这是整个演示的核心段落，需要放慢节奏，充分展示——**

**画面：** 确保在学习助手页 → 在输入框中打字："神经网络训练时那个从后往前算梯度的过程是什么？" → 点击发送 → **重点展示右侧「执行追踪」面板** — 节点根据意图按需执行（约 8 个节点依次亮起完成），未执行的节点保持暗色 → 展开紫色的「学习导航」展示多跳依赖路径 → 回复以打字机流式输出，包含公式和代码高亮 → 点击反馈标签

**口播：**

> "接下来进入系统的核心——学习助手。学生问了一个非常自然的、不含术语的问题：『神经网络训练时那个从后往前算梯度的过程是什么？』
>
> 注意看右侧面板——系统基于 **多智能体协作引擎**，正在实时编排执行。这里有一个重要设计：**不会机械地执行全部节点，而是根据意图按需路由**。系统先做意图识别，判断出学生在问「反向传播」，然后只加载与该意图相关的节点——可以看到大约 8 个节点依次亮起完成：**意图理解 → 记忆召回 → 证据检索 → 证据评估 → 生成回答 → 画像提取**。其余暗色节点是当前场景不需要的（如练习评估节点、路径规划节点），它们被智能跳过而非报错。这种按需路由机制让每次对话响应更快、更精准。
>
> 展开紫色的「学习导航」——这是本系统区别于普通 AI 助手的核心能力。系统找到了反向传播需要的前置知识：**链式法则、梯度下降、多层神经网络**。这不是凭空生成的回答，而是从知识图谱中真实检索出的依赖路径。**每一条回答都可追溯到图谱关系和讲义来源**，从机制上抑制了幻觉。
>
> 回复正在以打字机效果流式输出，支持数学公式渲染和代码高亮。在答案还未完整呈现时，学生已经能看到『正在检索证据』『正在评估质量』——智能体的工作过程完全透明。
>
> 对每条回复，学生可以一键点击反馈标签——『有帮助』『没看懂』『太深了』『例子不够』等共 8 种。系统通过自适应学习算法增量更新行为画像，下一次回答会自动调整讲解深度和风格，真正实现『越用越懂你』。"

**操作要点：**
- 发送问题后，鼠标指向「执行追踪」面板，跟随节点执行过程移动
- 口播中明确提到"按需路由，约 8 个节点执行"，避免让观众误以为剩余节点是失败的
- 展开「学习导航」时动作稍慢，让观众看清多跳依赖路径的内容
- 打字机效果输出时，鼠标可以指向回复中的公式和代码区域

### ⏱ 段落④：知识图谱（3:30 - 4:05 / 35秒 / 占比8%）

**画面：** 导航切换到「知识图谱」→ 搜索框输入"反向传播" → 节点高亮 → 放大展示颜色图例 → 点击节点查看详情，展示前置依赖边 → 缩放图谱展示全景

**口播：**

> "接下来看看底层的知识图谱。系统内置了 **370 个知识点节点、1067 条关系边**，覆盖《机器学习》11 章、50 个核心知识点。
>
> 搜索「反向传播」，节点颜色标识你的掌握状态——请注意图例：
> - 🔴 **红色** = 薄弱点，需要重点加强
> - 🟢 **绿色** = 已掌握
> - 🔵 **深蓝色** = 正在学习中
> - 🟠 **橙色** = 需要复习
> - ⚪ **灰色** = 待学习/推荐学习
>
> 点击节点可以查看详情和前置依赖路径，刚才证据面板中的多跳分析就来源于此。右上角的仪表盘还显示了总体的学习进度和个性化推荐起点。"

**操作要点：** 搜索后稍等图谱渲染完成。口播颜色图例时，鼠标配合指向对应颜色的节点，让观众视觉上对应上。

### ⏱ 段落⑤：资源生成（4:05 - 4:50 / 45秒 / 占比11%）

**画面：** 导航切换到「资源生成」→ 搜索选择"反向传播"知识点 → 勾选全部 6 类资源类型 → 点击生成 → 展示进度面板（6 个智能体并发生成）→ 生成完成后展示各类资源 → 重点展开思维导图，演示可交互展开/折叠 → 切换到知识中心，展示历史资源列表

**口播：**

> "进入资源生成。选择一个知识点，系统会并发生成 **6 类学习资源**：讲解文档、可交互思维导图、练习题、视频脚本、代码案例，以及教学图片。
>
> 注意进度面板——6 个智能体同时在后台工作。这里有个关键设计叫 **智能自修复机制**：每类资源走『生成→按类型校验→如果失败，把原因喂回 AI 修复→再校验』的链路。AI 输出本来是不可控的，但这套机制把概率性生成变成了工程上可靠的交付。失败后还会明确标注原因，支持单独重试。
>
> 生成完成后，思维导图支持在线展开和折叠，代码案例可以直接复制运行，全部资源自动保存到知识中心，随时回看。"

**操作要点：** 生成等待期间自然口播，不要冷场。展开思维导图时点击几个分支展示交互性。

---

### ⏱ 段落⑥：练习测试 + 评估报告（4:50 - 5:30 / 40秒 / 占比10%）

**画面：** 导航切换到「练习」→ 选择知识点"反向传播" → 选择「快速练习」模式 → 做 2-3 道选择题（依次点击选项）→ 提交 → 显示评分结果 → 导航到「评估」→ 展示雷达图、柱状图、饼图

**口播：**

> "进入练习模块。选择知识点和练习模式，系统支持快速练习、章节测验、错题重做和复习推荐四种模式。
>
> 选择题即时判分，简答、案例分析和编程题由 AI 评分并给出详细的针对性反馈。评分结果按知识点回写掌握度，并且带有完整的证据链——你可以追溯到是『哪次练习、哪道题』影响了你的掌握度评分。
>
> 转到评估报告页——雷达图直观展示各知识点的掌握度，柱状图看章节分布，饼图看不同题型的表现。薄弱点一目了然。"

**操作要点：** 做题时快速选择答案，提交后短暂停顿等待评分结果展示。

---

### ⏱ 段落⑦：学习路径（5:30 - 6:00 / 30秒 / 占比7%）

**画面：** 导航切换到「学习路径」→ 展示拓扑排序生成的推荐列表 → 鼠标划过几个推荐的节点，查看推荐理由

**口播：**

> "学习路径基于诊断结果，用 **智能路径规划算法** 生成个性化推荐队列。每个推荐节点都附带 **推荐理由**——例如『该节点是你目标『神经网络』的前置知识，且上次练习正确率仅 40%』。
>
> 系统标注了 5 种推荐类型：补弱、复习、目标推进、新知学习、遗忘唤醒。逐节点可解释，学生不仅知道『学什么』，还知道『为什么学这个』。"

**操作要点：** 鼠标悬停在某个推荐项上，展示 recommendation reason 的弹出提示。

---

### ⏱ 段落⑧：学习成长（6:00 - 6:30 / 30秒 / 占比7%）

**画面：** 导航切换到「学习成长」→ 展示日历热力图 → 向下滚动展示事件时间轴 → 再向下展示遗忘预警列表

**口播：**

> "最后看看学习成长。日历热力图记录了每一天的学习强度——颜色越深，学得越多。事件时间轴可以追溯每一次对话、练习和资源生成记录，全部持久化保留。
>
> 底部是遗忘预警，基于艾宾浩斯遗忘曲线自动检测该复习的知识点，帮助学生合理安排复习节奏，对抗遗忘。"

**操作要点：** 滚动页面时速度适中，让每个区域都有 2-3 秒的展示时间。

### ⏱ 段落⑨：总结收尾（6:30 - 7:00 / 30秒 / 占比7%）

**画面：** 画面切回学习助手页的全景，停留 → 最后逐渐黑屏

**口播：**

> "总结一下 EduGraph-Agent 的三个核心价值：
>
> **有依据**——每条回答都追溯到知识图谱的依赖路径与讲义来源，从机制上抑制幻觉，让学生可以核查 AI 的依据。
>
> **有路径**——从诊断到路径规划，从资源生成到练习评估，从画像回写到成长追踪，形成完整的学习闭环。
>
> **有回响**——三层建模让系统越用越懂你，且每一次适应调整都对学生可见。
>
> EduGraph-Agent——让每一次学习，有依据、有路径、有回响。谢谢观看！"

**操作要点：** 语速稍放缓，每个价值点之间停顿 1 秒，营造收尾感。

## 三、录制前检查清单

### 环境确认
- [ ] 后端已启动：`curl http://127.0.0.1:8000/health` → `{"status":"ok","neo4j":"ok"}`
- [ ] 前端已启动：浏览器可访问 `http://127.0.0.1:5174`
- [ ] Neo4j 连接正常，图谱数据已导入
- [ ] LLM API Key 有效，余额充足
- [ ] 演示账号 `demo@edugraph.local` 已有完整画像和历史数据

### 演示数据准备
- [ ] 画像对话已有 5 条以上记录，完整度 ≥ 80%
- [ ] 至少一次完整的学习助手对话历史
- [ ] 练习记录 ≥ 3 次，涉及不同知识点
- [ ] 资源生成记录 ≥ 2 次，知识中心有可展示内容
- [ ] 学习成长页有时间轴和热力图数据

### 录制设备
- [ ] 屏幕分辨率设置为 1920×1080
- [ ] 麦克风测试正常，无杂音
- [ ] 使用无痕/隐私模式打开浏览器
- [ ] 关闭所有通知弹窗
- [ ] 录制区域内无敏感/个人标签页

### 排练建议
1. **先通读 2-3 遍口播**，熟悉节奏和转场
2. **完整走一遍操作流程**，确认每个页面的数据都正常加载
3. **计时排练**：用手机计时器跑一遍，确保不超 7 分钟
4. **如果某环节出问题**：宁可跳过（如资源生成失败），不要卡壳——在口播中自然过渡到下一个功能
5. **核心段（学习助手）建议单独排练 3 遍以上**，这部分占 33% 时长，是评分的关键

## 四、应急预案

| 异常场景 | 口播替代方案 |
|----------|-------------|
| AI 响应慢（>5 秒） | "系统正在深度检索知识图谱的多跳依赖路径……"（自然过渡） |
| 资源生成部分失败 | "个别资源若生成失败，系统会明确标注原因并支持单独重试——这是智能自修复机制的工程保障" |
| 图谱搜索无结果 | 直接导航找一个已知存在的节点，如"梯度下降" |
| 练习评分延迟 | 提前截图评分结果备用，或口播时简述评分机制 |
| 页面空白/崩溃 | 刷新页面，用演示账号重新登录；口播时简短说明 |
