## 1. 测试环境

| 项 | 配置 |
|----|------|
| 操作系统 | Windows 10/11（亦支持 macOS / Linux） |
| Python | 3.11+ |
| Node.js | 18+ |
| Neo4j | 5.x，`bolt://localhost:7687`，已导入图谱数据（370 节点 / 1067 边） |
| 后端 | `uvicorn app.main:app --port 8000` |
| 前端 | `npm run dev`（http://127.0.0.1:5173） |
| LLM | OpenAI / DeepSeek 兼容端点，有效 API Key |
| 浏览器 | Chrome / Edge 最新版 |

## 2. 测试策略

| 层级 | 方式 | 覆盖 |
|------|------|------|
| 自动化端到端冒烟测试 | `backend/tests/test_e2e_smoke.py` | 健康检查 → 图谱查询 → 画像 → 诊断 → GraphRAG → 资源生成 → 知识中心 完整闭环 |

## 3. 自动化冒烟测试

### 3.1 运行方法

```bash
# 前置：Neo4j + 后端已启动，.env 配置有效
cd backend
python tests/test_e2e_smoke.py
```

### 3.2 覆盖用例

| # | 用例 | 断言 |
|---|------|------|
| 1 | 健康检查 `GET /health` | `status == ok`，Neo4j 连接正常 |
| 2 | 图谱节点查询 `GET /api/graph/node/ml_kmeans` | 200，uid 匹配，含 label/node_type |
| 3 | 子图查询 `GET /api/graph/subgraph/ml_kmeans?depth=1` | 200，返回 nodes/relationships |
| 4 | 画像初始化 `POST /api/profile/init` | 画像创建成功 |
| 5 | 诊断推荐 `POST /api/diagnosis/recommend` | 返回拓扑有序推荐列表 |
| 6 | GraphRAG 证据 `GET /api/graphrag/evidence` | EvidencePackage 结构完整 |
| 7 | 资源生成 `POST /api/agents/generate-resources` | 多类资源返回 |
| 8 | 知识中心 `GET /api/agents/resource-center` | 生成记录已持久化 |

输出格式：逐项 `[PASS]/[FAIL]`，全部通过则退出码 0。

## 4. 接口测试用例（手工）

后端启动后打开 Swagger UI：`http://127.0.0.1:8000/docs`。

### 4.1 认证

| 用例 | 步骤 | 预期 |
|------|------|------|
| 注册 | POST `/api/auth/register`（email/username/password） | 201，返回用户信息 |
| 重复注册 | 同邮箱再次注册 | 4xx，明确错误信息 |
| 登录 | POST `/api/auth/login` | 200，返回 access_token + refresh_token |
| 错误密码 | 错误密码登录 | 401 |
| 演示登录 | POST `/api/auth/demo` | 200，返回 access_token |
| token 刷新 | POST `/api/auth/refresh` | 200，新 access_token |

### 4.2 学习助手 / GraphRAG

| 用例 | 步骤 | 预期 |
|------|------|------|
| SSE 流式对话 | POST `/api/assistant/stream`，问"帮我讲解反向传播" | 事件序列 run_started→node_started→…→final_response→persisted |
| 多跳依赖 | `GET /api/graphrag/evidence?uid=ml_backpropagation` | 含 multi_hop_summary 依赖路径 |
| 自然语言证据查询 | POST `/api/graphrag/query`，"反向传播" | EvidencePackage 含图+语义双通道证据 |
| 宽泛查询降级 | 问"机器学习的应用" | 降级匹配提示 + 备选建议，不硬答 |
| 反馈提交 | POST `/api/assistant/feedback`（标签"没看懂"） | 200，behavior profile 更新 |
| 越权反馈 | 用他人 student_id 提交 | 4xx 拒绝 |

### 4.3 画像 / 练习 / 诊断

| 用例 | 步骤 | 预期 |
|------|------|------|
| 画像对话构建 | POST `/api/profile/chat` 描述学习背景 | 画像字段更新，完整度上升 |
| 画像手动编辑 | PATCH `/api/profile/{id}` | 字段更新并记录时间线 |
| 创建练习会话 | POST `/api/exercises/sessions`（4 种模式各一次） | 会话创建，题目返回 |
| 提交答案 | POST `/api/exercises/sessions/{id}/submit` | 选择题即时判分；简答/代码/案例 LLM 评分 + 反馈 |
| 画像回写 | 提交整轮练习后查 `GET /api/profile/{id}` | node_mastery 更新，带证据链 |
| 路径推荐 | POST `/api/diagnosis/recommend` | 拓扑有序，每节点含推荐理由 |
| 成长时间轴 | GET `/api/profile/{id}/timeline` | 事件按日聚合，可追溯 |

### 4.4 资源生成

| 用例 | 步骤 | 预期 |

## 6. 异常与降级测试

| 用例 | 操作 | 预期 |
|------|------|------|
| LLM 不可用 | 移除/改错 LLM_API_KEY | 后端返回明确 unavailable，不伪装智能体；基础功能（图谱查询）可用 |
| Neo4j 断连 | 停止 Neo4j | `/health` 报 neo4j 异常；相关接口明确报错不崩溃 |
| 证据不足 | 查询图谱外冷门问题 | 降级匹配 + 备选建议 |
| 资源生成超时 | 网络受限时生成 | 部分成功返回，失败项标注 |
| SSE 中断 | 对话中刷新页面 | 会话恢复，历史完整 |
