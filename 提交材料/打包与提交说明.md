## 3. 打包前必做（⚠️ 安全与规范）

### 3.1 密钥脱敏（最高优先级）

- [ ] **删除或清空 `backend/.env`**——内含 4 个真实 API Key（DeepSeek / 中转端点 / SiliconFlow / 讯飞）；
- [ ] **删除或清空根目录 `.env`**——同样内含真实的 DeepSeek 和 SiliconFlow Key（config.py 会同时读取两处 .env，两处都要清）；
- [ ] 打包前把用到的 Key 在对应平台 **轮换/作废** 一次更稳妥（已进过 git 历史的密钥视为泄露）；
- [ ] 确认包内只保留 `提交材料/部署配置/.env.example` 模板。

### 3.2 排除以下目录/文件

| 排除项 | 原因 |
|--------|------|
| `参考教材/` | 有版权的教材 PDF，体积大且有侵权风险 |
| `frontend/node_modules/` | 依赖可由 `npm install` 重建，体积巨大 |
| `**/__pycache__/`、`*.pyc` | 编译缓存 |
| `.git/` | 版本库（如需展示提交历史可另行说明） |
| `backend/.env`、根目录 `.env` | 真实密钥（见 3.1，两处都有） |
| `data/edugraph.db`、`data/chroma/` | 含测试用户数据；交付干净初始状态，首次运行自动重建 |
| `data/generated_assets/` | 运行期生成物 |
| `*.log`、`tmp_*` | 临时文件 |

### 3.3 建议的打包命令（PowerShell，在项目根目录的上一级执行）

```powershell
# 用 git archive 最干净（自动应用 .gitignore 规则，但注意 提交材料/ 需已提交）
cd g:\data_space\EduGraph-Agent
git add 提交材料; git commit -m "docs: 添加提交材料"
git archive -o ..\EduGraph-Agent-提交包.zip HEAD

# 或者手动压缩时排除（7-Zip 示例）
7z a ..\EduGraph-Agent-提交包.zip . `
  -xr!node_modules -xr!__pycache__ -xr!.git -xr!参考教材 `
  -x!backend\.env -x!data\edugraph.db -xr!data\chroma -xr!data\generated_assets
```

### 3.4 打包后自检

- [ ] 解压到全新目录，按 `提交材料/部署配置/部署说明.md` 从零走一遍，能启动、能登录、能对话；
- [ ] 全文搜索包内 `sk-`、`API_SECRET`，确认无真实密钥残留；
- [ ] 文档内相对路径链接可达；
- [ ] 总体积合理（预计 < 100 MB）。
