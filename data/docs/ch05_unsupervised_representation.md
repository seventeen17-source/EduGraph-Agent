# 第 5 章 无监督学习与特征表示

## 章节定位

本章对应《机器学习》聚类、降维与度量学习、特征选择与稀疏学习等章节，用于补充无标签数据分析和特征表示能力。

## 优先级

P0。正式收集时优先补 K-Means、聚类、降维和特征选择，其他内容可后补。

## 5.1 聚类

```text
knowledge_node_id: ml_clustering
source_ids: book_zhou_ml, book_prml_bishop, docs_sklearn_user_guide
title: 聚类的无监督分组思想
importance: 0.85
content: 【核心概念】聚类是在没有标签的情况下，根据样本之间的相似性或距离关系，把数据划分成若干组，每组称为一个簇。它关注的是数据内部结构，而不是预测预先给定的类别。【直觉理解】如果面对一堆没有标注的用户行为记录，聚类就像先根据相似购买习惯把用户自然分群，再由人解释每个群体的含义。【与前置知识的关系】需要先理解监督学习与无监督学习的区别，以及样本、特征和距离的含义；它是 K-Means、层次聚类和密度聚类的基础。【典型误区】聚类结果不等于真实类别，簇的语义通常需要结合业务解释；不同距离度量和参数设置可能得到不同划分。【应用场景】用户分群、异常检测、图像分割、数据探索和预处理阶段常用聚类。
keywords: 聚类, 无监督学习, 相似度, 簇, 距离度量, 数据探索
```

## 5.2 K-Means

```text
knowledge_node_id: ml_kmeans
source_ids: book_zhou_ml, book_prml_bishop, book_hands_on_ml_3e_zh, docs_sklearn_user_guide
title: K-Means 的簇中心迭代过程
importance: 0.90
content: 【核心概念】K-Means 是划分式聚类算法，需要预先指定簇数 K。算法反复执行两步：把每个样本分配给最近的簇中心，再用簇内样本均值更新中心，直到分配结果或目标函数趋于稳定。【直觉理解】K-Means 像先放几个临时集合点，让每个样本去最近的集合点报到，再根据报到的人群重新移动集合点，循环直到位置基本不变。【与前置知识的关系】需要先理解聚类、均值和距离度量；度量选择会直接影响“最近”的含义。【典型误区】K 值不能随便设，过小会合并不同结构，过大可能把自然簇切碎；K-Means 也不保证找到真实簇，且对初始中心和异常点较敏感。【应用场景】客户分群、图像颜色量化、文档聚类和无监督数据探索常用 K-Means。
keywords: K-Means, 簇中心, 迭代, 距离度量, K值, 划分聚类
```

## 5.3 降维

```text
knowledge_node_id: ml_dimensionality_reduction
source_ids: book_zhou_ml, book_prml_bishop, book_mml_deisenroth, docs_sklearn_user_guide
title: 降维的压缩与表示思想
importance: 0.80
content: 【核心概念】降维是把高维数据映射到较低维空间，同时尽量保留对任务有用的信息。线性降维中，PCA 通过寻找方差较大的投影方向来保留主要变化结构；非线性降维则更关注复杂流形或局部邻域关系。【直觉理解】高维数据像信息很多但难以观察的物体，降维相当于选择一个合适视角投影，让主要结构更容易看清，同时减少噪声和计算负担。【与前置知识的关系】需要先理解向量、矩阵、投影和特征表示；它与聚类、可视化和特征工程密切相关。【典型误区】降维不是简单删除特征，PCA 等方法会构造新的低维表示；维度越低也不一定越好，过度压缩会丢失关键信息。【应用场景】可视化、高维特征压缩、降噪、加速训练和聚类前处理常用降维。
keywords: 降维, PCA, 投影, 特征表示, 可视化, 高维数据
```

## 5.4 度量学习

```text
knowledge_node_id: ml_metric_learning
source_ids: book_zhou_ml, book_prml_bishop, book_mml_deisenroth
title: 度量学习如何定义相似性
importance: 0.70
content: 【核心概念】度量学习关注如何学习适合任务的距离或相似度，使同类或相似样本更近，不同类或不相似样本更远。相比固定使用欧氏距离，度量学习会根据数据和任务调整特征空间中的距离关系。【直觉理解】“像不像”并没有唯一标准：比较房子时面积和地段可能重要，比较图片时颜色和纹理可能重要。度量学习就是让模型学会当前任务下哪些差异更值得重视。【与前置知识的关系】需要先理解向量空间、距离、聚类和分类；它会影响 KNN、K-Means、检索和表征学习的表现。【典型误区】距离公式不是中立的，特征尺度和权重会改变相似性判断；学到的度量也可能过拟合训练任务。【应用场景】人脸识别、图像检索、推荐系统、聚类前处理和相似案例匹配中常用度量学习思想。
keywords: 度量学习, 距离, 相似度, 表示学习, 欧氏距离, 检索
```

## 5.5 特征选择

```text
knowledge_node_id: ml_feature_selection
source_ids: book_zhou_ml, book_hands_on_ml_3e_zh, docs_sklearn_user_guide
title: 特征选择的降噪与解释作用
importance: 0.80
content: 【核心概念】特征选择是从原始特征中挑选对任务更有用的子集，以减少冗余、噪声和计算成本，并提升模型解释性。常见方法包括过滤式、包裹式和嵌入式：过滤式先按统计指标筛选，包裹式结合模型表现搜索特征子集，嵌入式在训练过程中完成选择。【直觉理解】建模像准备答题材料，信息越多不一定越好；无关或重复材料会干扰判断，特征选择就是保留更有价值的证据。【与前置知识的关系】需要先理解过拟合、性能度量和监督学习任务；它与正则化、稀疏学习和模型解释密切相关。【典型误区】特征选择不是越少越好，删掉关键信息会欠拟合；在全数据上先做选择再划分测试集会造成数据泄漏。【应用场景】高维数据、文本分类、生物信息、风控建模和可解释模型构建中常用特征选择。
keywords: 特征选择, 过滤式, 包裹式, 嵌入式, 过拟合, 数据泄漏
```

## 5.6 稀疏学习

```text
knowledge_node_id: ml_sparse_learning
source_ids: book_zhou_ml, book_pumpkin_ml, book_hands_on_ml_3e_zh
title: 稀疏学习与少量关键特征
importance: 0.70
content: 【核心概念】稀疏学习希望模型只依赖少量关键特征或参数，使许多权重变为零或接近零。L1 正则化是常见方式，它在优化目标中惩罚参数绝对值，常能产生稀疏解，从而同时达到特征选择和模型简化的效果。【直觉理解】如果一个任务真正只由少数因素决定，稀疏学习就像让模型少说废话，只保留最有解释力的因素，而不是让每个特征都参与预测。【与前置知识的关系】需要先理解特征选择、正则化和损失函数；它是高维建模和可解释学习的重要工具。【典型误区】稀疏不等于一定更准确，过强稀疏约束会丢失有用信息；L1 与 L2 的效果不同，L1 更容易把参数压到零。【应用场景】文本特征、高维医学数据、Lasso 回归、压缩表示和可解释模型中常用稀疏学习。
keywords: 稀疏学习, L1正则化, 特征选择, Lasso, 高维数据, 可解释性
```
