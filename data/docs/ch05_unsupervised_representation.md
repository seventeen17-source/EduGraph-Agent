# 第 5 章 无监督学习与特征表示

## 章节定位

本章对应《机器学习》聚类、降维与度量学习、特征选择与稀疏学习等章节，用于补充无标签数据分析和特征表示能力。

## 优先级

P0。正式收集时优先补 K-Means、聚类、降维和特征选择，其他内容可后补。

## 5.1 聚类

```text
knowledge_node_id: ml_clustering
source_ids: book_zhou_ml, book_prml_bishop, docs_sklearn_user_guide
title: 聚类的无监督分组思想
importance: 0.85
content: 【核心概念】聚类（Clustering）是在没有标签的情况下，根据样本之间的相似性或距离关系，把数据划分成若干组（簇）的任务。每组称为一个簇（Custer），簇内样本彼此相似，簇间样本差异较大。与监督学习的核心区别在于：聚类关注的是数据内部结构和自然分组，而不是预测预先给定的类别标签。相似性的度量通常使用距离函数（如欧氏距离、马氏距离、余弦相似度），不同的距离度量会直接影响聚类结果。【直觉理解】如果面对一堆没有标注的用户行为记录，聚类就像先根据相似购买习惯把用户自然分群，再由人解释每个群体的含义——机器先告诉我们"这里自然分成三类"，人再赋予这三类"高消费型""观望型""冲动型"等语义。【与前置知识的关系】需要先理解监督学习与无监督学习的根本区别（是否有标签），以及样本、特征、距离度量的含义；聚类是 K-Means、层次聚类、DBSCAN 和谱聚类等多种具体算法的基础。【典型误区】聚类结果不等于真实类别——聚类算法不知道真实标签，簇的语义含义通常需要人工解释。不同距离度量和参数设置会得到完全不同的划分，不存在唯一"正确"的聚类结果。【常用算法】K-Means（划分式）、层次聚类（自底向上或自顶向下）、DBSCAN（基于密度）、谱聚类（基于图论）。【应用场景】用户分群（电商、社交媒体）、异常检测（识别离群点）、图像分割、文档聚类（主题发现）、数据探索和预处理阶段常用聚类。
keywords: 聚类, 无监督学习, 相似度, 簇, 距离度量, K-Means, 层次聚类, DBSCAN
```

## 5.2 K-Means

```text
knowledge_node_id: ml_kmeans
source_ids: book_zhou_ml, book_prml_bishop, book_hands_on_ml_3e_zh, docs_sklearn_user_guide
title: K-Means 的簇中心迭代过程
importance: 0.90
content: 【核心概念】K-Means 是划分式聚类算法，需要预先指定簇数 K。算法反复执行两步：把每个样本分配给最近的簇中心，再用簇内样本均值更新中心，直到分配结果或目标函数趋于稳定。目标函数是所有样本到其所属簇中心的欧氏距离平方之和，迭代过程使这一指标单调下降（但不保证全局最优）。K-Means 本质上是在高维特征空间中做 Voronoi 图划分，每个簇中心对应一个区域。【直觉理解】K-Means 像先放几个临时集合点，让每个样本去最近的集合点报到，再根据报到的人群重新移动集合点，循环直到位置基本不变。整个过程相当于反复执行”投票—重新指定召集人”的民主过程，最终各群体收敛到一个稳定配置。【与前置知识的关系】需要先理解聚类任务本身（无监督分组的意义）、均值作为中心代表的思想，以及欧氏距离度量。如果把 K-Means 看成 EM 算法的特例，理解会更加深刻：E 步是分配样本，M 步是更新中心。【典型误区】K 值不能随便设，过小会合并不同结构，过大可能把自然簇切碎；K-Means 也不保证找到真实簇，且对初始中心选择和异常点较敏感——异常点会把簇中心拉偏，使结果不稳定；随机初始化多次取最优是常见缓解手段。K-Means 对特征尺度敏感，如果不同维度量纲差异大，需要先做标准化。【应用场景】客户分群、图像颜色量化（将 1677 万种颜色压缩到 K 种代表色）、文档主题初探、无监督数据探索和作为其他聚类算法的初始化步骤。sklearn.cluster.KMeans 是 Python 中最常用的实现接口。
keywords: K-Means, 簇中心, 迭代, 距离度量, K值, 划分聚类, EM思想, Voronoi
```

## 5.3 降维

```text
knowledge_node_id: ml_dimensionality_reduction
source_ids: book_zhou_ml, book_prml_bishop, book_mml_deisenroth, docs_sklearn_user_guide
title: 降维的压缩与表示思想
importance: 0.80
content: 【核心概念】降维（Dimensionality Reduction）是把高维数据映射到较低维空间，同时尽量保留对任务有用的信息。降维方法分为线性和非线性两类。线性降维中，主成分分析（PCA）通过寻找方差最大的正交投影方向来保留数据的主要变化结构；线性判别分析（LDA）则是有监督的投影方法，寻找使类间方差最大、类内方差最小的方向。t-SNE 和 UMAP 是常用的非线性降维方法，更关注保持局部邻域结构，适合可视化。【直觉理解】高维数据像信息很多但难以直观观察的多面体，降维相当于选择一个合适的视角投影，让主要结构更容易看清，同时减少噪声和计算负担。PCA 像是找出数据中变化最大的几个"主方向"，把这些方向作为新坐标轴，从而用更少的维度描述数据的核心变化。【与前置知识的关系】需要先理解向量、矩阵、投影和特征表示的数学基础；降维与聚类、可视化、特征工程密切相关——降维后的表示常作为下游任务的输入特征。【典型误区】降维不是简单删除某些特征，PCA 等方法会构造全新的低维表示（主成分是原始特征的线性组合）。降维后维度越低不一定越好——过度压缩会丢失关键判别信息，导致下游任务性能下降。PCA 需要数据标准化，特征尺度差异大时会主导方差方向。【关键公式】PCA 的目标：max_w Var(w^Tx) subject to ||w|| = 1。第一主成分 w 是协方差矩阵最大特征值对应的特征向量。【应用场景】高维数据可视化（t-SNE/UMAP）、高维特征压缩加速模型训练、噪声过滤、聚类和分类前处理。sklearn.decomposition.PCA 是最常用的实现。
keywords: 降维, PCA, 投影, 特征表示, 可视化, t-SNE, UMAP, 主成分分析, LDA
```

## 5.4 度量学习

```text
knowledge_node_id: ml_metric_learning
source_ids: book_zhou_ml, book_prml_bishop, book_mml_deisenroth
title: 度量学习如何定义相似性
importance: 0.70
content: 【核心概念】度量学习（Metric Learning）关注如何学习适合特定任务的距离或相似度度量，使同类或相似样本在特征空间中距离更近，不同类或不相似样本距离更远。相比固定使用欧氏距离或余弦相似度，度量学习会根据数据分布和下游任务自动调整特征空间中各维度的相对重要性。典型方法包括 LMNN（Large Margin Nearest Neighbor）、NCA（Neighbourhood Components Analysis）和基于对比学习的方法（如 Siamese 网络）。【直觉理解】”像不像”并没有唯一标准——比较房子时面积和地段可能更重要，比较图片时颜色和纹理可能更重要。度量学习就是让模型学会当前任务下哪些维度的差异更值得重视，而不是把所有维度同等对待。【与前置知识的关系】需要先理解向量空间、距离度量（欧氏距离、余弦相似度）、KNN 和分类任务；度量学习直接影响 KNN 分类、K-Means 聚类、图像检索和人脸识别的效果。【典型误区】距离公式不是中立的——不同特征的尺度和权重会完全改变相似性判断，学到的度量也可能过拟合训练数据的标签分布。度量学习学到的不一定是人类直觉的”相似性”，而是最适合任务目标的相似性定义。【应用场景】人脸识别（FaceNet 的三元组损失）、图像检索（以图搜图）、推荐系统（协同过滤的度量扩展）和聚类前处理都常用度量学习思想。
keywords: 度量学习, 距离度量, 相似度, LMNN, NCA, Siamese网络, 对比学习, 人脸识别
```

## 5.5 特征选择

```text
knowledge_node_id: ml_feature_selection
source_ids: book_zhou_ml, book_hands_on_ml_3e_zh, docs_sklearn_user_guide
title: 特征选择的降噪与解释作用
importance: 0.80
content: 【核心概念】特征选择（Feature Selection）是从原始特征中挑选对目标任务最有价值的子集，目标是减少冗余和噪声特征、降低计算成本、提升模型泛化能力和可解释性。与特征工程不同，特征选择不创造新特征，而是从已有特征中做减法。常见三类方法：过滤式（Filter）用统计指标（如相关系数、卡方检验、互信息）独立评估每个特征与标签的相关性，不依赖模型，速度快但不考虑特征间的组合效应；包裹式（Wrapper）用模型在候选特征子集上的表现来评估（如递归特征消除 RFE），效果更好但计算量大；嵌入式（Embedded）在模型训练过程中自动完成特征选择（如带 L1 正则化的逻辑回归、随机森林的特征重要性），兼具速度和效果。【直觉理解】建模像准备论文材料，不是所有材料都同等重要——无关或重复材料会干扰评审（模型）的判断，特征选择就是保留最有价值的证据，去掉干扰项。【与前置知识的关系】需要先理解过拟合（特征选择可以缓解）、性能度量（用模型表现评估特征子集）和监督学习任务；它与正则化（L1 嵌入式选择）、稀疏学习（稀疏解等于自动特征选择）和模型解释密切相关。【典型误区】特征选择不是越少越好——删除关键信息会欠拟合，特征子集质量比数量更重要。在全数据上做完特征选择后再划分训练/测试集，会造成数据泄漏（测试集信息提前进入了选择过程），正确做法是先划分再用训练集做特征选择。【应用场景】高维数据（基因数据、文本 TF-IDF 特征）、生物信息学、风控建模和可解释模型构建中常用特征选择。sklearn.feature_selection 提供了多种实现。
keywords: 特征选择, 过滤式, 包裹式, 嵌入式, 过拟合, 数据泄漏, 递归特征消除, L1正则化
```

## 5.6 稀疏学习

```text
knowledge_node_id: ml_sparse_learning
source_ids: book_zhou_ml, book_pumpkin_ml, book_hands_on_ml_3e_zh
title: 稀疏学习与少量关键特征
importance: 0.70
content: 【核心概念】稀疏学习（Sparse Learning）希望模型只依赖少量关键特征或参数，使大多数权重变为零或接近零，从而达到同时做特征选择、降低模型复杂度和提升可解释性的效果。L1 正则化（Lasso）是最常见的稀疏手段，它在优化目标中惩罚参数绝对值之和，使许多参数被精确压缩到零，从而产生稀疏解。几何直觉：等值线与 L1 范数约束（菱形）相交时，更容易落在坐标轴上（对应某些参数恰好为零）。L2 正则化（Ridge）使参数趋近于零但不会精确为零，所以 L2 不是稀疏的。Elastic Net 结合 L1 和 L2 的优点。【直觉理解】如果一个任务真正只由少数几个因素决定，稀疏学习就像让模型"少说废话"，只保留最有解释力的因素参与预测，而不是让每个特征都凑一份影响力。【与前置知识的关系】需要先理解特征选择（稀疏是自动特征选择的一种方式）、正则化（稀疏学习的实现手段）和损失函数；它是高维建模、模型压缩和可解释学习的重要工具。【典型误区】稀疏不等于一定更准确——过强的稀疏约束会导致关键特征也被删除，反而欠拟合。L1 和 L2 的效果有本质区别：L1 产生稀疏解，L2 只让参数变小但不归零。稀疏表示（让神经元激活稀疏）和稀疏权重（让参数稀疏）是不同的概念，前者是指网络输出的表示，后者是指模型参数本身。【应用场景】高维特征数据（基因数据、文本）、Lasso 回归用于同时做预测和特征选择、模型压缩（剪枝）、可解释 AI 中常用稀疏学习。
keywords: 稀疏学习, L1正则化, L2正则化, Lasso, Ridge, ElasticNet, 高维数据, 可解释性
```
