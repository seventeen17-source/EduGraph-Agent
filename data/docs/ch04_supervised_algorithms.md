# 第 4 章 经典监督学习算法

## 章节定位

本章主要对应《机器学习》决策树、支持向量机、贝叶斯分类器和集成学习等章节，用于补充线性模型之外的经典监督学习方法。

## 优先级

P0。正式收集时优先补充决策树、SVM、贝叶斯分类器和集成学习；随机森林可作为集成学习的工程案例。

## 4.1 决策树

```text
title: 决策树的递归划分思想
importance: 0.85
content: 【核心概念】决策树通过一系列特征判断逐步划分样本空间，内部节点表示划分条件，叶节点给出预测结果。训练时通常选择能让划分后样本更“纯”的特征，例如信息增益、增益率或基尼指数。【直觉理解】决策树像一份逐步提问的规则表：先问最有区分力的问题，再在每个分支继续追问，直到样本足够明确或达到停止条件。【与前置知识的关系】需要先理解监督学习、分类任务和性能度量；它又是随机森林、Boosting 等集成方法的重要基学习器。【典型误区】树越深不一定越好，过深容易记住训练样本细节而过拟合；剪枝不是破坏模型，而是提升泛化能力的常用手段。【应用场景】决策树适合需要可解释规则的分类和回归任务，也常作为集成学习的基础组件。
keywords: 决策树, 信息增益, 基尼指数, 剪枝, 可解释性
```

## 4.2 支持向量机

```text
knowledge_node_id: ml_svm
source_ids: book_zhou_ml, book_lihang_statistical_learning, book_pumpkin_ml, docs_sklearn_user_guide
title: 支持向量机的最大间隔思想
importance: 0.80
content: 【核心概念】支持向量机通过寻找分类间隔最大的超平面来进行分类，真正决定边界的是离超平面最近的一小部分样本，即支持向量。对非线性问题，核函数可以在不显式构造高维特征的情况下计算高维空间中的相似度。【直觉理解】如果要在两类点之间画一条分界线，SVM 不只追求分对训练样本，还希望分界线两侧留出尽可能宽的安全距离。【与前置知识的关系】需要先理解线性代数、线性分类和几何间隔；LDA、逻辑回归和 SVM 都是线性分类思想的不同延伸。【典型误区】核函数不是越复杂越好，复杂核可能过拟合；SVM 也不是只能用于小数据，但在大规模数据上训练成本需要认真考虑。【应用场景】SVM 适合中小规模分类、文本分类、高维稀疏特征和需要清晰间隔解释的任务。
keywords: 支持向量机, 间隔, 超平面, 核函数, 支持向量
```

## 4.3 贝叶斯分类器

```text
knowledge_node_id: ml_bayesian_classifier
source_ids: book_zhou_ml, book_lihang_statistical_learning, book_prml_bishop
title: 贝叶斯分类器的概率决策思想
importance: 0.75
content: 【核心概念】贝叶斯分类器基于贝叶斯定理，把先验概率和样本特征出现的条件概率结合起来，计算不同类别的后验概率，并选择后验概率最大的类别。朴素贝叶斯进一步假设特征在给定类别下条件独立，从而大幅简化估计。【直觉理解】贝叶斯分类像根据证据更新判断：先有对类别的初始估计，再观察样本特征，根据这些证据调整每个类别的可信程度。【与前置知识的关系】需要先理解概率、条件概率、随机变量和分类任务；它为概率图模型、生成式分类和文本分类打基础。【典型误区】朴素贝叶斯的“朴素”不是模型粗糙无用，而是指条件独立假设较强；即使假设不完全成立，它在许多任务上仍有实用效果。【应用场景】垃圾邮件过滤、文本分类、简单医学辅助判断和小数据概率分类中常使用贝叶斯方法。
keywords: 贝叶斯, 条件概率, 后验概率, 朴素贝叶斯, 分类
```

## 4.4 集成学习

```text
knowledge_node_id: ml_ensemble_learning
source_ids: book_zhou_ml, book_hands_on_ml_3e_zh, docs_sklearn_user_guide
title: 集成学习的组合提升思想
importance: 0.80
content: 【核心概念】集成学习通过组合多个基学习器来获得更好的整体表现。Bagging 通过对数据重采样训练多个模型并平均或投票，主要降低方差；Boosting 逐步关注前一轮难以处理的样本，常用于降低偏差；Stacking 则用新的模型学习如何组合多个模型输出。【直觉理解】一个模型可能判断失误，但多个差异化模型共同投票，往往比单个模型更稳。关键不是简单堆数量，而是让基学习器既有一定能力又有差异。【与前置知识的关系】需要先理解决策树、偏差-方差和模型评估；随机森林、AdaBoost、梯度提升树都是集成思想的具体实现。【典型误区】集成模型不一定总比单模型好，如果基学习器高度相似或都很差，组合收益有限；集成也会增加训练和解释成本。【应用场景】表格数据建模、分类竞赛、风险评分和工程预测任务中经常使用集成学习。
keywords: 集成学习, Bagging, Boosting, Stacking, 基学习器, 投票
```

## 4.5 随机森林

```text
knowledge_node_id: ml_random_forest
source_ids: book_zhou_ml, book_hands_on_ml_3e_zh, docs_sklearn_user_guide
title: 随机森林如何降低决策树方差
importance: 0.75
content: 【核心概念】随机森林是 Bagging 思想的典型应用，它训练多棵决策树，并在训练每棵树时引入样本随机性和特征随机性，最后通过投票或平均得到预测结果。单棵树可能方差很高，而多棵差异化树的组合通常更稳定。【直觉理解】一棵决策树像一个人的判断，容易受训练数据细节影响；随机森林像让一组看过不同样本、关注不同特征的人共同投票，从而减少偶然偏差。【与前置知识的关系】需要先理解决策树、Bagging 和偏差-方差权衡；它是从单模型走向集成模型的直观案例。【典型误区】随机森林不是简单复制同一棵树，随机采样和随机选特征是制造差异的关键；树越多通常更稳定，但计算成本和解释难度也会上升。【应用场景】随机森林常用于表格数据分类与回归、特征重要性分析和强基线模型构建。
keywords: 随机森林, 决策树, Bagging, 投票, 特征随机性, 方差
```
