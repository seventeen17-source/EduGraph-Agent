## 1.1 基本术语

```text
knowledge_node_id: ml_basic_terms
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 机器学习基本术语的核心概念
importance: 1.0
content: 【核心概念】机器学习的基本术语用于描述“数据如何进入模型”。一批记录构成数据集，每条记录是一个样本或示例；描述样本的维度称为属性或特征，具体取值是属性值，多个特征组成特征向量，并落在样本空间中。带有结果信息的样本拥有标记，学习或训练就是利用训练集寻找从输入到输出的规律。【直觉理解】可以把一条西瓜记录看成一个样本，色泽、根蒂、敲声是特征，好瓜或坏瓜是标记；模型要做的是从许多样本中总结判断规则。【与前置知识的关系】这是课程入口，先分清样本、特征、标签和模型，才能继续理解监督学习、损失函数和评估指标。【典型误区】不要把样本和特征混为一谈；特征数量增加也不必然提升效果，噪声特征可能干扰学习。【应用场景】这些术语用于描述分类、回归、聚类等任务，也是后续讲解训练流程、泛化和模型评估的共同语言。
keywords: 样本, 特征, 标签, 模型, 训练, 数据集, 分类, 回归, 聚类
```

## 1.2 监督学习与无监督学习

```text
knowledge_node_id: ml_supervised_unsupervised
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 监督学习与无监督学习的核心区别
importance: 1.0
content: 【核心概念】周志华《机器学习》第1章明确区分：监督学习使用带有标记信息的训练数据，目标是学到从输入到输出的映射，包括分类（离散标记）和回归（连续标记）。无监督学习处理无标记数据，目标是发现数据内在结构，如将样本划分为若干簇（聚类）。介于两者之间还有半监督学习和强化学习，其中强化学习通过与环境交互获得延迟的奖惩信号来调整策略。【直觉理解】Andrew Ng的ML Specialization提供了一个经典类比：监督学习像有标准答案的习题集，通过对照答案来改进解题方法；无监督学习像面对一堆没有标签的照片，让机器自己根据相似性分成几堆。【与前置知识的关系】必须先掌握样本、特征、标记这三个基本术语的含义，才能理解有标记和无标记的核心区别。本节点是学习线性回归、逻辑回归、决策树等具体监督算法以及聚类、降维等无监督方法的入口。【典型误区】混淆聚类和分类——聚类没有预定义的类别标签，簇的语义含义通常需要人工解释。不区分半监督和无监督——半监督仍有少量标签。【应用场景】监督学习主导了实际中的绝大多数ML应用（图像识别、语音识别、推荐系统）；无监督学习常用于数据探索、用户分群、异常检测。
keywords: 监督学习, 无监督学习, 分类, 回归, 聚类, 标记, 半监督学习, 强化学习
```

## 1.3 训练集、验证集与测试集

```text
knowledge_node_id: ml_dataset_split
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 训练集、验证集与测试集的划分逻辑
importance: 0.95
content: 【核心概念】周志华《机器学习》第2章模型评估与选择中详细讨论了数据划分问题。训练集用于训练模型、学习参数；验证集用于超参数调优和模型选择；测试集仅在最终阶段用于评估泛化性能。三者必须保持互斥：测试集中的样本绝不能出现在训练过程中，否则泛化误差的估计会偏乐观。实际应用中，数据集通常被划分成训练集和测试集，再从训练集中分出验证集。【直觉理解】Andrew Ng将这种划分类比为考试：训练集是平时作业和练习题（可反复做），验证集是模拟考（检验学习效果并调整复习策略），测试集是正式考试（考完才知道真实水平，不能提前看题）。如果让测试集数据泄漏到训练阶段，就相当于考试前偷看了答案。【与前置知识的关系】必须先理解样本、标记等基本术语。本节点是理解泛化、过拟合、交叉验证、留出法等上层评估概念的基石。【典型误区】认为验证集可有可无——没有验证集时通常将测试集当验证集反复使用，导致模型选择过程泄漏测试集信息。认为划分比例是固定的——数据量大时验证和测试集可适当减小比例，小样本时需用交叉验证。【应用场景】常用划分比例为6:2:2或8:1:1。小样本场景下交叉验证优于单次留出法划分。
keywords: 训练集, 验证集, 测试集, 数据划分, 留出法, 交叉验证, 泛化
```

## 1.4 泛化能力

```text
knowledge_node_id: ml_generalization
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 泛化能力的含义与重要性
importance: 0.95
content: 【核心概念】周志华《机器学习》第2章明确将泛化定义为学得模型适用于新样本的能力。训练集上的误差称为经验误差，新样本上的误差称为泛化误差。机器学习的根本目标不是最小化经验误差，而是最小化泛化误差。由于新样本的标记未知，泛化误差无法直接计算，通常用测试集上的测试误差来近似。【直觉理解】Andrew Ng反复强调：如果一个模型在训练集上表现完美但一上线就失效，建模工作就完全白费了。衡量模型好坏的唯一标准是它在从未见过的数据上的表现。训练集上的性能是手段，泛化性能才是目的。【与前置知识的关系】必须先理解训练集和测试集的划分，才能区分经验误差和泛化误差。本节点是理解过拟合、欠拟合、偏差-方差分解和模型选择策略的理论前提。【典型误区】把测试误差等同于真实泛化误差——测试误差只是泛化误差的近似估计。训练误差持续下降意味着模型越来越好——当训练误差低但测试误差高时反而是危险信号。【应用场景】泛化能力是评估一切机器学习模型的核心指标，直接决定了模型是否能部署上线。
keywords: 泛化, 泛化误差, 经验误差, 测试误差, 训练误差, 过拟合
```

## 1.5 过拟合与欠拟合

```text
knowledge_node_id: ml_overfitting_underfitting
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 过拟合与欠拟合的判断与应对
importance: 1.0
content: 【核心概念】周志华《机器学习》第2章将过拟合定义为学习器把训练样本自身的一些特点当作了所有潜在样本都会具有的一般性质，导致泛化性能下降。欠拟合则是对训练样本的一般性质尚未学好。周志华特别强调一个关键结论：过拟合是机器学习中无法彻底避免的问题，只能缓解。原因在于机器学习面对的是NP难问题，且训练样本只是真实分布的一个有限采样，其中不可避免地包含噪声和采样偏差。【直觉理解】过拟合相当于学生把考卷上每道题的专有特征（如排版位置、字体大小）误当作普适解题规律背了下来，同一套题换个排版就做不出来了。欠拟合相当于连最基本的知识框架都没掌握。Andrew Ng建议通过观察训练误差和验证误差的差距来判断：两者都高是欠拟合，训练误差低但验证误差显著偏高是过拟合。【与前置知识的关系】必须先理解泛化能力的概念。本节点是正则化、偏差-方差分析、早停、Dropout等防过拟合技术的理论基础。【典型误区】认为训练准确率高等于模型好——这是混淆了经验误差和泛化误差。认为加更多数据一定能解决过拟合——增加数据量有助于缓解过拟合但不能根除。【应用场景】过拟合诊断是调参的第一步。解决方法包括：增加训练数据、简化模型、正则化、早停、Dropout（神经网络）、数据增强。
keywords: 过拟合, 欠拟合, 泛化, 经验误差, 模型复杂度, 正则化, 偏差, 方差
```

## 1.6 模型评估方法

```text
knowledge_node_id: ml_evaluation_methods
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 模型评估方法的核心思想与适用场景
importance: 0.90
content: 【核心概念】周志华《机器学习》第2章系统介绍了三种划分训练集和测试集的方法。留出法将数据集直接划分为互斥的两部分，其中训练集用于学习、测试集用于评估。交叉验证法将数据集等分为k个子集，每次用k-1个子集训练、1个子集测试，轮流k次取均值，留一法是k=m的特例。自助法通过有放回采样构造训练集，样本在m次采样中始终不被采到的概率约为0.368，因此约36.8%的样本可作为测试集。【直觉理解】留出法相当于用一次考试评判学生，简单但结果受试卷难易波动影响大。交叉验证相当于安排多场考试取平均分，评估更稳定但更费时。自助法相当于从有限题库中有放回地抽题反复练习，再用剩下的题目测试，适合样本极少时使用。【与前置知识的关系】必须先理解训练集和测试集的基本划分和泛化误差的概念，才能理解不同评估方法的适用场景。【典型误区】留出法中训练集比例越高越好——训练集过大会使测试集评估不稳定。自助法可普遍替代交叉验证——自助法改变了数据分布，会引入估计偏差。留一法一定最好——虽然评估准确，但计算开销大且结果方差可能较大。【应用场景】k折交叉验证（k=5或10）是中小数据集的标准评估方法。留出法适合大数据集。自助法适合样本数极少的情况。
keywords: 留出法, 交叉验证, 自助法, 留一法, 模型评估, 泛化, k折
```

## 1.7 性能度量

```text
knowledge_node_id: ml_performance_metrics
source_ids: book_zhou_ml, course_andrew_ng_ml_specialization
title: 性能度量的常用指标与选择策略
importance: 0.90
content: 【核心概念】周志华《机器学习》第2章对性能度量做了详细阐述。回归任务最常用均方误差。分类任务的核心指标来自混淆矩阵：查准率（Precision）指预测为正的样本中真正为正的比例；查全率（Recall）指真实正例中被正确找出的比例；二者通常相互制约。F1是查准率和查全率的调和平均，Fβ是加权调和平均（β>1时侧重查全率）。ROC曲线以真正例率（TPR）为纵轴、假正例率（FPR）为横轴，AUC是ROC曲线下面积，衡量模型对样本的排序质量。【直觉理解】查准率和查全率的权衡在生活中很常见：搜索引擎返回的结果中真正相关的比例是查准率，数据库中所有相关文档被搜出来的比例是查全率。想要两者都高通常需要更好的模型或接受某种折中。【与前置知识的关系】必须先理解模型评估方法与测试集的含义，才能在测试集上计算这些指标。【典型误区】在类别不平衡时仅看准确率——若99%的样本为负，全判为负即可获99%准确率但无实际价值。混淆查准率与查全率——查准率关注找到的是否正确，查全率关注该找的是否都找全了。【应用场景】垃圾邮件过滤偏重查准率（减少误判），疾病筛查偏重查全率（减少漏诊），搜索引擎偏重前k个结果的查准率。
keywords: 准确率, 查准率, 查全率, F1, AUC, ROC, 混淆矩阵, 均方误差
```

## 1.8 偏差与方差

```text
knowledge_node_id: ml_bias_variance
source_ids: book_zhou_ml, book_pumpkin_ml, course_andrew_ng_ml_specialization
title: 偏差-方差分解与权衡
importance: 0.85
content: 【核心概念】周志华《机器学习》第2章给出了泛化误差的偏差-方差分解：泛化误差等于偏差的平方加方差加噪声。偏差衡量学习算法期望预测与真实结果的偏离程度，刻画拟合能力；方差衡量相同规模训练集变动导致的学习结果变化程度，刻画数据扰动的影响；噪声是问题本身的难度，构成泛化误差的理论下界。偏差主导意味着欠拟合，方差主导意味着过拟合，这被称为偏差-方差窘境。【直觉理解】周志华在书中用射箭类比解释了偏差和方差的区别，南瓜书补充了详细的数学推导过程。偏差高意味着瞄准本身就偏了，不管打多少次都偏离靶心；方差高意味着每次打的位置散布很开但平均可能很接近靶心。模型复杂度很低时偏差主导（打不准），复杂度上升时方差逐渐变为主导（打很准但散布太大）。【与前置知识的关系】必须先理解泛化误差和过拟合与欠拟合，才能理解为什么泛化误差可以这样分解。本节点是正则化和集成学习（Bagging主要降方差，Boosting主要降偏差）的理论基础。【典型误区】认为偏差和方差可以同时降到极低——受噪声项和样本有限性的限制，两者存在此消彼长的权衡关系。认为方差高只有坏处——在集成学习中适当利用不同模型的方差反而可以提高整体性能。【应用场景】偏差-方差分析用于诊断模型问题：高偏差→增加模型复杂度；高方差→增加数据或正则化。
keywords: 偏差, 方差, 噪声, 泛化误差, 偏差方差权衡, 欠拟合, 过拟合
```

## 1.9 正则化

```text
knowledge_node_id: ml_regularization
source_ids: book_zhou_ml, book_pumpkin_ml, course_stanford_cs229
title: 正则化的核心思想与常用形式
importance: 0.90
content: 【核心概念】周志华《机器学习》第2章讨论了正则化作为缓解过拟合的核心策略。正则化在经验风险（训练集上的损失）基础上增加一个对模型复杂度的惩罚项，构成结构风险。优化目标变为：经验风险加正则化项乘以正则化系数λ。L1正则化对参数绝对值求和，容易产生稀疏解，部分参数被压缩为0从而起到特征选择作用。L2正则化对参数平方求和，使所有参数趋向于较小的值，但不为零，可改善数值稳定性。λ越大惩罚越重，模型越简单；λ越小越接近原始模型。【直觉理解】周志华将正则化类比为奥卡姆剃刀原则在机器学习中的具体实现：在同样能解释数据的模型中，更简单的模型泛化能力往往更好。南瓜书补充了L1产生稀疏解的几何直觉：等值线与菱形约束区域相交时容易落在坐标轴上（某些参数恰好为0）。【与前置知识的关系】必须先理解过拟合（正则化的直接动机）、损失函数（正则化项的载体）以及偏差-方差分解（正则化通过略微增加偏差来大幅降低方差）。【典型误区】正则化只会降低模型表现——适当的正则化虽略微增加训练误差，但提升泛化性能。L1和L2只是名字不同——L1更擅长产生稀疏解实现特征选择，L2对异常值更敏感但数值更稳定。【应用场景】L2正则化广泛用于线性回归（岭回归）、逻辑回归和深度学习（权重衰减）。L1正则化常用于高维数据中需要自动特征选择的场景（Lasso回归）。Elastic Net结合了L1和L2的优点。
keywords: 正则化, L1, L2, 过拟合, 结构风险, 奥卡姆剃刀, 稀疏, 岭回归, Lasso
```
