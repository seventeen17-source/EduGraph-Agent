每章一个 Markdown 文件，放在 `data/docs/` 下：

```text

```

### 1.3.2 文件内部格式

每个知识点的讲义文本**用 `` ```text `` 代码块包裹**，块内为 `key: value` 格式。

导入脚本 `Scripts/import_to_neo4j.py` 中的 `parse_doc_chunks()` 会解析这些块并生成 `DocumentChunk` 节点 + `SUPPORTS` 边。

**模板**（一个章节文件中可以放多个 `` ```text `` 块）：

````markdown
# 第 X 章 章节标题

## X.Y 知识点名称

```text
knowledge_node_id: ml_xxx
source_ids: book_zhou_ml, course_stanford_cs229
title: 知识点名称的核心概念
importance: 0.9
content: |
  【核心概念】
  {一句话说清这个知识点是什么。}

  【直觉理解】
  {不用公式，用生活类比或可视化描述解释核心思想。每个类比控制在 2-3 句。}

  【与前置知识的关系】
  {指明为什么先学 A 才能学这个。建议引用具体的知识点名称或 uid。}

  【典型误区】
  {1-2 个学生最容易犯的错误。与 knowledge_points.json 中的 common_misconceptions 保持一致。}

  【应用场景】
  {1-2 句说明这个知识点在什么任务或什么模型中被直接使用。}

keywords: 关键词1, 关键词2, 关键词3
```
````

### 1.3.3 字段说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `knowledge_node_id` | ✅ | 挂载的目标知识点 uid，必须能在 `knowledge_points.json` 中找到 |
| `source_ids` | ✅ | 英文字段逗号分隔，每个值必须能在 `data/sources/sources.json` 中找到 |
| `title` | ✅ | 该 chunk 的简短标题，用于检索和前端展示 |
| `importance` | ✅ | 0-1 浮点数。核心概念 0.9~1.0；补充说明 0.6~0.8；扩展材料 0.4~0.6 |
| `content` | ✅ | 多行文本。使用 `\|` 续写，支持 Markdown 内联格式 |
| `keywords` | ✅ | 英文逗号分隔，用于规则检索增强 |

## 1.4 内容写作标准

### 1.4.1 每篇 chunk 长度

**200~400 字中文。** 低于 200 字证据密度不够，超过 500 字切成 2 个 chunk。

### 1.4.2 写作要求

1. **先读教材，然后合上书写。** 理解后用自己的语言表述，不使用教材原文。
2. **写完后回查。** 如果某段与教材原文相似度超过 70%，必须改写。
3. **每个 chunk 至少标注 1 个 `source_ids`。** 即使只有一本教材作为参考来源也要标注。
4. **不要写完整代码。** 代码示例放在 `data/code_cases/` 下。
5. **不要写完整数学推导。** 公式用行内 LaTeX（如 `$w = w - \eta \nabla L(w)$`）简要表示即可。
6. **类比必须与知识点本质一致。** 不要用似是而非的比喻。如果不确定，先不写类比。

### 1.4.3 示例：完整的真实 chunk 块

以下是可以直接参考的写作范例（以 `ml_gradient_descent` 为例）：

````markdown
## 3.2 梯度下降

```text
knowledge_node_id: ml_gradient_descent
source_ids: book_zhou_ml, course_stanford_cs229
title: 梯度下降的核心思想与直觉理解
importance: 0.95

content: |
  【核心概念】
  梯度下降是一种迭代优化算法，通过沿损失函数梯度的反方向更新模型参数，逐步降低损失函数值。每次迭代中，算法计算当前参数处的梯度方向（损失函数上升最快的方向），然后沿该方向的反方向移动一小步，步长由学习率控制。

  【直觉理解】
  想象你站在一座山上，想走到最低点。每次你环顾四周找到最陡的下坡方向，沿这个方向走一步。重复这个过程，你会逐渐接近山谷底部。在这里，梯度就是“最陡方向”，学习率就是“步长”。步子太大可能直接跨过最低点甚至摔下山，步子太小则要走很久。

  【与前置知识的关系】
  理解梯度下降需要先掌握损失函数（作为被优化的目标）和梯度与最优化的基础概念（理解了梯度的数学含义才能理解参数更新公式）。在线性回归中，梯度下降被用来最小化均方误差损失，它是理解反向传播和 SGD 的关键前置。

  【典型误区】
  1. “学习率越大收敛越快”——实际上过大的学习率会导致参数更新跨越最优点，在损失曲面上震荡甚至发散。
  2. “梯度下降总能找到全局最优”——在非凸损失函数中，梯度下降可能停在局部最优或鞍点。

  【应用场景】
  梯度下降及其变体（SGD、Mini-batch、Adam 等）是训练绝大多数机器学习模型的基础优化方法，包括线性回归、逻辑回归和深度神经网络。

keywords: 梯度, 学习率, 损失函数, 参数更新, 收敛, 局部最优
```
````

### 2.3.4 `assesses` 和 `also_assesses` 的区别

| 字段 | 含义 | 举例 |
|---|---|---|
| `assesses` | 主题目考察的核心知识点 | 梯度下降概念题 → `["ml_gradient_descent"]` |
| `also_assesses` | 该题同时也能反映的其他知识点掌握情况 | 同一题也间接测到了损失函数 → `["ml_loss_function"]` |

每条 `assesses` 会生成一条 Neo4j `ASSESSES` 边。`also_assesses` 不会自动生成边，但它会被 `rank_nodes_by_profile()` 用于个性化排序。

## 2.4 题型选择指南

### 选择题
适合检验概念理解和辨析能力。每个核心知识点至少 1-2 题。
- 选项要有迷惑性，B/C/D 不能是随机拼凑
- 错误选项要对应 `misconception_tags` 中的误区

### 简答题
适合检验深层理解。每个核心知识点至少 1 题。
- 需要填写 `rubric` 和 `answer.key_points`
- `grading.method` 设为 `rubric`
- `auto_gradable` 通常为 false（当前 LLM 批改）

### 编程题
适合检验实操能力。优先给 P0 核心节点、代码友好型知识点。
- 需要填写 `test_cases`
- `grading.method` 设为 `unit_test`
- 提供 `starter_code`

### 案例分析题
适合综合知识考察。不必每章都有，但 P0 章节至少 1-2 题。
- 适合过拟合、集成学习、聚类等应用型知识点

## 3.3 写入规范

### 3.3.1 存放位置

`data/code_cases/` 下，每个知识点一个 `.py` 文件。命名建议：

```text
data/code_cases/
  gradient_descent_demo.py
  linear_regression_demo.py
  logistic_regression_demo.py
  knn_demo.py
  decision_tree_demo.py
  kmeans_demo.py

```

### 3.3.2 文件头部元数据

导入脚本 `parse_code_case()` 通过正则解析 `.py` 文件的头部注释：

```python
"""最小梯度下降演示。
Related node: ml_gradient_descent
Source IDs: book_hands_on_ml
"""
```

**重要约定**：
- `Related node:` 后的值是知识点 `uid`，会生成 `PRACTICES` 边
- `Source IDs:` 后的值是英文字段逗号分隔，指向 `sources.json`

### 3.3.3 代码内容要求

1. **30~60 行**，太长不好理解，太短没有价值
2. **自包含**——复制到本地能直接跑
3. **关键行有中文注释**
4. **末尾有实验任务**——引导学生修改参数、观察变化
5. **使用 sklearn / NumPy / Matplotlib** 等常见库，避免冷门依赖
6. **不依赖 GPU**

### 3.3.4 示例模板

```python
"""线性回归最小二乘与梯度下降对比演示。

Related node: ml_linear_regression
Source IDs: book_hands_on_ml, docs_sklearn_user_guide
"""

import numpy as np
from sklearn.linear_model import LinearRegression

# ---------- 1. 生成示例数据 ----------
# y = 2x + 1 + 噪声
np.random.seed(42)
X = np.linspace(0, 10, 100).reshape(-1, 1)
y = 2 * X.ravel() + 1 + np.random.randn(100) * 0.5

# ---------- 2. sklearn 直接拟合 ----------
model = LinearRegression()
model.fit(X, y)
print(f"sklearn 拟合结果: w={model.coef_[0]:.3f}, b={model.intercept_:.3f}")

# ---------- 3. 从零实现批量梯度下降 ----------
w, b = 0.0, 0.0
lr = 0.01
for epoch in range(100):
    y_pred = w * X.ravel() + b
    # 计算梯度
    dw = -2 * np.mean((y - y_pred) * X.ravel())
    db = -2 * np.mean(y - y_pred)
    w -= lr * dw
    b -= lr * db
    if epoch % 20 == 0:
        loss = np.mean((y - y_pred) ** 2)
        print(f"epoch {epoch:3d}: loss={loss:.4f}, w={w:.3f}, b={b:.3f}")

print(f"梯度下降拟合结果: w={w:.3f}, b={b:.3f}")

# ---------- 实验任务 ----------
# 1. 把 learning_rate 改成 0.1，观察收敛速度变化
# 2. 把 learning_rate 改成 1.0，观察是否震荡
# 3. 把数据量从 100 改成 10，观察拟合效果
```
