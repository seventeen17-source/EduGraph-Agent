# EduGraph-Agent 启动服务说明

## 1. 前置条件

- Neo4j 已启动，默认 `bolt://localhost:7687`（账号 neo4j / edugraph123）
- LLM API Key 可用（OpenAI 或 DeepSeek 兼容端点）
- Python 3.11+ / Node.js 18+

## 2. 配置环境

在 `backend/.env` 中配置：

```env
LLM_API_KEY=你的key
LLM_MODEL=gpt-4o-mini
LLM_BASE_URL=你的base_url（使用代理时必填，如 https://api.deepseek.com/v1）
LLM_PROVIDER=openai（或 deepseek）

EMBEDDING_API_KEY=你的key（可与 LLM_API_KEY 相同）
EMBEDDING_MODEL=text-embedding-3-small

NEO4J_URI=bolt://localhost:7687
NEO4J_USERNAME=neo4j
NEO4J_PASSWORD=edugraph123
NEO4J_DATABASE=neo4j

DATABASE_URL=sqlite+aiosqlite:///../data/edugraph.db
```

## 3. 安装依赖

```powershell
# 后端
cd backend
pip install -r requirements.txt

# 前端
cd ../frontend
npm install
```

## 4. 启动服务

终端 1 — 后端：
```powershell
cd backend
uvicorn app.main:app --reload --port 8000
```

终端 2 — 前端：
```powershell
cd frontend
npm run dev
```

验证后端：
```
http://127.0.0.1:8000/health → {"status":"ok","neo4j":"ok"}
```

打开前端：
```
http://127.0.0.1:5173
```

## 7. 常见问题

### 前端 Network Error
检查后端是否启动：`http://127.0.0.1:8000/health`

### 搜索知识点无结果
确认 Neo4j 已启动且数据已导入。搜索支持模糊匹配名称、别名、关键词和 UID。

### npm 执行策略错误
Windows PowerShell 使用 `npm.cmd` 而非 `npm`。

### LLM 不可用
检查 `.env` 中 `LLM_API_KEY` 配置和 API 余额。后端在无 LLM 时返回明确 unavailable 状态，不会伪装智能体。
