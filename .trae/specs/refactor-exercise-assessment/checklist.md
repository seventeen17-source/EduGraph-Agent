# Checklist

## P0: 统一练习来源池

- [ ] 后端 `POST /exercises/search` 端点可返回多来源题目（题库题 + AI生成题 + 错题）
- [ ] 搜索结果每条记录包含 `source_type` 字段（knowledge_base / ai_generated / mistake / recommended）
- [ ] 前端练习页搜索框输入 "K-Means" 后返回多来源结果
- [ ] 前端来源筛选 Tab（全部/题库题/AI生成/我的错题/推荐复习）可正确过滤
- [ ] 从搜索结果进入作答并提交后，练习记录正常保存
- [ ] 在助手中生成 K-Means 练习题后，不通过 resource_id 跳转，直接搜索 K-Means 也能找到刚生成的题

## P0: AI备课台移除默认内容

- [ ] 直接打开 `/resources`（无 query 参数），输入框为空
- [ ] 页面显示引导文案，不显示默认神经网络问题
- [ ] 资源类型默认不全选
- [ ] 选择"做题巩固"模式时，只选中 exercise 类型
- [ ] 选择"全套学习包"时，才选中全部类型
- [ ] 从知识图谱/助手/知识中心带参数进入时，query 正确预填

## P0: 助手生成题后显示"开始作答"按钮

- [ ] 用户让助手生成题后，聊天区不直接展开所有题干和选项
- [ ] 消息卡片显示"立即开始作答"按钮（主按钮）
- [ ] "查看资源"按钮仍存在但为次要操作
- [ ] 点击"开始作答"后进入练习页并加载本次生成的题
- [ ] 作答提交后进入练习记录和画像更新链路

## P0: 练习模式选择

- [ ] 练习页显示模式选择器（自由练习/正式测验/错题复习/诊断评估）
- [ ] 正式测验模式提交前不能查看答案和解析
- [ ] 自由练习模式可以查看提示，但 used_hint 记录为 true
- [ ] 提交时 mode 字段正确传递到后端
- [ ] 画像更新时能区分"独立答对"和"看提示后答对"
- [ ] ExerciseAttempt 模型包含 mode、viewed_answer、grading_method 字段

## P0: 后端按题型评分

- [ ] choice 题型使用规则评分（grading_method=rule）
- [ ] short_answer 题型使用 LLM 评分（grading_method=llm_rubric），同义表达不误判
- [ ] coding 题型使用结构化评分（grading_method=code_execution）
- [ ] 前端非 choice 题型不传 is_correct，由后端评分
- [ ] 每道题保存 score、feedback、grading_method
- [ ] 前端展示后端返回的评分结果和反馈

## P0: 清理新用户 demo 数据

- [ ] 新注册用户进入评估页，不显示示例雷达图
- [ ] 新注册用户进入学习路径，不自动展示 demo 路径
- [ ] 新注册用户进入学习成长，不显示伪造图表
- [ ] 无画像数据时显示引导文案
- [ ] fallback/demo 内容不伪装成真实个性化结果

## P1: 体验增强

- [ ] 知识中心搜索支持资源正文、题干、代码搜索
- [ ] 练习记录详情显示 mastery_before/mastery_after 对比
- [ ] 练习记录支持按知识点、来源、题型、时间筛选
- [ ] 学习路径页面使用"推荐学习队列"而非线性编号
- [ ] 每个推荐节点显示推荐理由
- [ ] 学生侧不出现英文 enum 或调试字段（mode=llm、fallback_used 等）
- [ ] fallback 结果对学生以自然语言提示

## 最终验收主流程

- [ ] 新用户注册后完成基础画像
- [ ] 对助手说"帮我生成 10 道 K-Means 选择题"
- [ ] 助手不内联展示全部题目，显示"已生成 10 道题"
- [ ] 点击"立即开始作答"进入练习页，加载刚生成的题
- [ ] 提交后保存练习记录，错题进入错题本
- [ ] 画像中 K-Means 相关 mastery 更新
- [ ] 回到练习页搜索"K-Means"，能搜到刚才 AI 生成的题
- [ ] 回到知识中心，也能找到这次生成的资源
- [ ] 再次问助手"我刚才 K-Means 哪些地方错了"，助手能结合练习记录回答
